Laboratorio: Controllo Temperatura I2C

In questo esperimento avanzato collegheremo due Arduino via I2C. Il Master legge la temperatura (TMP36) e invia il dato grezzo allo Slave. Lo Slave controlla un Motore DC (tramite transistor) e dei LED di stato, garantendo che i dati siano sincronizzati perfettamente.

Lo Schema Elettrico (Con Transistor di Protezione)

SDA su A4, SCL su A5. Motore pilotato tramite Transistor NPN su Pin 9.

Esperimento 3: Master Sender & Slave Receiver

1. Codice Master (Sender)

Legge il sensore e invia un byte con segno.

#include <Wire.h>

const int pinSensore = A0;
float temperatura = 0.0;
int8_t segnale = 0; 

void setup() {
  Serial.begin(9600);
  Wire.begin(); // Master
}

void loop() {
  int lettura = analogRead(pinSensore);
  float volt = (lettura / 1024.0) * 5.0;
  temperatura = (volt - 0.5) * 100.0;
  
  // Cast fondamentale a int8_t
  segnale = (int8_t)temperatura;
  
  Wire.beginTransmission(0x55);
  Wire.write(segnale);
  Wire.endTransmission();
  
  Serial.print("Inviato: ");
  Serial.println(segnale);
  delay(100);
}

2. Codice Slave (Receiver)

Riceve, converte e pilota il motore in sicurezza.

#include <Wire.h>

const int pinTransistor = 9;   
const int ledFreddo = 12;      
const int ledCaldo = 13;       

// 'volatile' per variabili usate nell'ISR
volatile int8_t tempRicevuta = 0;
volatile bool datoNuovo = false;

void setup() {
  Serial.begin(9600);
  pinMode(pinTransistor, OUTPUT);
  pinMode(ledFreddo, OUTPUT);
  pinMode(ledCaldo, OUTPUT);
  
  Wire.begin(0x55); // Indirizzo Slave
  Wire.onReceive(riceviDati);
}

// Funzione ISR (Interrupt)
void riceviDati(int bytes) {
  if (Wire.available()) {
    tempRicevuta = (int8_t)Wire.read();
    datoNuovo = true;
  }
}

void loop() {
  if (datoNuovo) {
    Serial.print("RX: ");
    Serial.println((int)tempRicevuta);

    // Logica LED
    if (tempRicevuta > 30) {
      digitalWrite(ledCaldo, HIGH);
      digitalWrite(ledFreddo, LOW);
    } else {
      digitalWrite(ledCaldo, LOW);
      digitalWrite(ledFreddo, HIGH);
    }

    // Logica Motore con Transistor
    // < 20¬∞C: Spento. > 50¬∞C: Max.
    int pwm = map(tempRicevuta, 20, 50, 0, 255);
    analogWrite(pinTransistor, constrain(pwm, 0, 255));
    
    datoNuovo = false;
  }
}

Ecco le soluzioni tecniche adottate per evitare gli errori comuni (valori -40, motore bruciato, dati corrotti).

1. int8_t (Sincronizzazione) Abbiamo forzato entrambi gli Arduino a usare contenitori a 8 bit con segno. Questo impedisce che un valore negativo (es. -1) venga letto erroneamente come 255.
2. Transistor & Constrain Il motore richiede troppa corrente per il pin Arduino (40mA+). Il Transistor sposta il carico sui 5V. La funzione constrain() assicura che al pin PWM non arrivino mai numeri negativi, che causerebbero errori.
3. Volatile (Memoria) Poich√© tempRicevuta cambia improvvisamente dentro l'interrupt (ISR), volatile obbliga il processore a rileggere la RAM ogni volta, evitando di usare dati vecchi della cache.
