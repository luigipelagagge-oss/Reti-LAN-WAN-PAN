Esperimento 3: Controllo Remoto Temperatura

Un Arduino legge la temperatura e comanda un motore sull'altro Arduino usando il bus I2C (Libreria Wire.h). Rispetto alla Serial, qui usiamo indirizzi specifici (Master/Slave).

SENDER (Master)
#include <Wire.h> void setup() { Wire.begin(); // Avvia come Master Serial.begin(9600); } void loop() { int lettura = analogRead(A0); // Mappa la lettura (0-1023) in gradi (-40 a 125) // per comandare il motore del Receiver int segnale = map(lettura, 20, 358, -40, 125); // Invia al dispositivo indirizzo 0x55 Wire.beginTransmission(0x55); Wire.write(segnale); Wire.endTransmission(); Serial.print("Temperatura inviata: "); Serial.println(segnale); delay(200); }
RECEIVER (Slave 0x55)
#include <Wire.h> const int pinTransistor = 9; const int ledFreddo = 12; const int ledCaldo = 13; // int8_t per gestire correttamente i segni volatile int8_t tempRicevuta = 0; volatile bool datoNuovo = false; void setup() { Serial.begin(9600); pinMode(pinTransistor, OUTPUT); pinMode(ledFreddo, OUTPUT); pinMode(ledCaldo, OUTPUT); // Si registra sul bus con indirizzo 0x55 Wire.begin(0x55); // ISR: Funzione chiamata quando arrivano dati Wire.onReceive(riceviDati); Serial.println("RICEVITORE ALLINEATO"); } void riceviDati(int bytes) { if (Wire.available()) { tempRicevuta = (int8_t)Wire.read(); datoNuovo = true; } } void loop() { if (datoNuovo) { Serial.print("Temp: "); Serial.println((int)tempRicevuta); // 1. ‚ö° Teoria: Tensione vs Corrente

Abbiamo usato Serial e I2C (Tensione). Ma nell'industria reale, su lunghe distanze, si usa la Corrente.

Caratteristica RS232 / I2C (Tensione) Current Loop 4-20mA (Corrente)
Principio Legge i Volt (0V o 5V). Legge gli Ampere che scorrono nel filo.
Distanza Corta (metri). La resistenza del cavo fa calare i Volt. Lunga (km). La corrente rimane costante ovunque nel loop.
Diagnostica Se leggi 0, potrebbe essere "LOW" o filo rotto. Se leggi 0mA, il filo √® rotto (il minimo √® 4mA).
